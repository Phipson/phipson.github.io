Voxel Modeling in 3D Space

Overview

Timeline

4 Weeks (1 Month)
August 2019

Involved members

Phipson (Me)/ Prototype Developer

Project Content

ARKit on iPad; LeapMotion gesture library; Voxel-modeling; Product Analysis and case studies


As part of a 4-week exploratory research activity at Apple's Applied Research Team, I collaborated with engineers and designers at Apple to investigate methods users could create personalized assets in AR.

Currently, one of the challenges when using Reality Composer is that casual designers and end users do not get the opportunity to directly build and create their own 3D models to interact with. Instead, they must use pre-existing models for their composition, or design assets using external software. To improve the workflow and creative capabilities of Reality Composer, I investigated ways users could maximize the capabilities of voxel modeling to create models using free-form, expressive hand gestures.

In the process of designing this interface, I used the AR capabilities on the latest iPad. I also captured 3D Hand Gestures by combining the ARKit environment with LeapMotion sensors.


(As my work is under Apple's NDA, I am not allowed to describe the project in detail. However, my work was demoed live at internal Open House events, and videos of my work have been showcased across different teams at Apple)

Design Process

Profiles and Use Cases

To explore different 3D modeling, the group came together for multiple brainstorming sessions examining different possible use cases and design goals. After narrowing down the scope of our project, I designed the following user profiles repesenting the different scenarios users would want to design assets for Reality Compoosers, these user profiles are both from personal experiences using 3D modeling CAD software, existing literature surrounding CAD software, as well as other group members' experiences with 3D modeling. Here, we are assuming that end users have some design experience and thus are capable of designing their own virtual content in 3D space.

From these user profiles I identified the following design challenges most common with existing 3D designers:

1. Finding a balance between expressively using free-form hand gestures to design, while also having a high level of precision and accuracy when using those gestures to add detail.
2. Having a dynamic design workflow that optimizes both the ability to generate large volumes quickly, and also having fine-grained control with details of different portions of the model.

Examining these profiles in detail, I posited that using voxels as a foundation for modeling in 3D would adequately address the two design needs. This is primarily because:

1. Voxels offer a grid-like structure where even with coarse-grained gesture control and detection, shapes can be clearly defined thanks to mechanisms such as snapping.
2. Voxels also offer a method of representing large volumes and complex meshes, while also giving users control with small details, as illustrated later.

To examine existing workflows with voxels in a 2D interface, I referenced the following existing apps and design tools.

Minecraft: A first person role-playing game, Minecraft has also been used as a voxel modeling tool where players immersively create and modify voxels one by one in third virtual 3D environment. The ability to handle fine-grained detail by modifying 1 voxel at a time, and the immersive experience of doing so in first person are two elements I want to bring into my prototype.

MagicaVoxel: A third person tool for modeling voxelized models. By using click-and-drag gestures, this tool easily allows users to create large voxelized volumes such as mountains and buildings. This fast and efficient workflow is one thing I would like to emulate in my prototype.

Designing an Interface

Due to the time constraints of this research activity, I had to design a prototype with a limited set of design tools. Upon discussing with my research team and examining existing design tools, I defined 5 tools to best capture the basic functionality and design workflow that is possible with my voxel modeling tool.

In addition, I also modified an existing voxelization library such that users could voxelize an arbitrary mesh in the AR environment, and modify individual voxels using the above toolset. These tools formed the basis of my prototype.

Adding Gesture Input

Due to time constraints, in order to create a prototype before the exploratory activity ended, I reused an existing customized gesture library I previously made for a previous project involving visual logic programming in AR. Mapping the gestures to different functions, and also integrating it into the iPad AR environment, I was able to design a customized 2D and 3D interface where users could create voxels or voxelized meshes; erase individual voxels or groups of voxels; color voxels; navigate around the scene; and extrude voxelized surfaces.

Evaluation


The prototype was demoed, videoed, and presented at an internal Open House event, showcasing the possibilities of voxel modeling in 3D space. Additionally, code for my project was shared with members of the App Development team at Apple. Although I was not able to design and conduct formal user studies for the project, evaluating the interface, I would consider making the following modifications if I had time.

Performance Optimization

Due to memory limitations, the voxelization can lead to potential frame rate drops. Furthermore, as the number of voxels propagate, there are noticeable performance issues with both memory and speed. As a result, to scale this design tool, we would need to find ways of optimizing the data and memory to maintain the same level of performance even with a high number of voxels.

Menu Scalability

One of the noticeable limitations with 2D interfaces for CAD software is the fact that it is difficult to navigate the plethora of tools available. In a 3D setting, I would like to investigate ways of designing a menu such that tools are easily identifiable, so that users do not need to spend excessive amounts of time finding their desired tool.
